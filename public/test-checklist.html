API Testing Checklist

This checklist covers all major user actions derived from the API routes and frontend functions. Use the checkboxes to mark off tested items.

1. Authentication & User Profile

  • Request Login Link: User requests a magic link for login.
    Endpoint: POST /api/auth/request-link
    Frontend: requestLoginLink(email, inviteToken?)
    Params: email (required), inviteToken (optional)
    Test: Valid email (success), invalid email (failure), with/without invite token.
  • Verify Login Link: User verifies the login link to authenticate.
    Endpoint: POST /api/auth/verify-link
    Frontend: verifyLoginLink(token)
    Params: token (required)
    Test: Valid token (success), invalid/expired token (failure).
  • Logout: User logs out.
    Endpoint: POST /api/auth/logout
    Frontend: logout()
    Params: None
    Test: Successful logout, session cleared.
  • Get User Profile: Fetch the logged-in user's profile data.
    Endpoint: GET /api/user/profile
    Frontend: getProfile(showToast?)
    Params: None
    Test: Authenticated user (success), unauthenticated (failure).
  • Update User Demographics: Update optional demographic info (e.g., age, zip code).
    Endpoint: POST /api/user/update-demographics
    Frontend: Not directly mapped (use updateProfile)
    Params: Demographic fields
    Test: Valid updates (success), invalid data (validation error).
  • Update Financial Profile: Update financial profile details.
    Endpoint: POST /api/user/update-financial-profile
    Frontend: Not directly mapped (use updateProfile)
    Params: Financial fields
    Test: Valid updates, permission checks for partners.
  • Dismiss Accounts Prompt: Dismiss the prompt to add accounts.
    Endpoint: POST /api/user/dismiss-accounts-prompt
    Frontend: dismissAccountsPrompt()
    Params: None
    Test: Successful dismissal, no further prompts.
  • Fresh Start (Reset Account): Reset user's account data (budgets, goals, etc.).
    Endpoint: DELETE /api/user/fresh-start
    Frontend: freshStart()
    Params: None
    Test: Owner-only (success), partner attempt (forbidden), data wiped.
